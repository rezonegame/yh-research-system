# research-intake

## 目标

将模糊研究需求转换为可执行规格，并进入 plan-first。

## 执行规范

1. 先推荐 1-2 个最相关 Skills。
2. 产出 requirements spec：
   - MUST / SHOULD / MAY
   - CLEAR / ASSUMED / BLOCKED
3. 给出阶段化计划与首个动作。
4. 指明建议输出路径（project path / templates / deliverables）。

## 输出模板

- 研究目标
- 研究问题
- 数据与证据来源
- 方法建议
- 风险与限制
- 下一步动作
